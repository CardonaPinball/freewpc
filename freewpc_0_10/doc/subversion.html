FreeWPC uses Subversion, a source control tool similar to cvs. The Subversion repository is available for anonymous read access. Developers who wish to deliver changes to the main project can send email to freewpc@oddchange.com and ask for an account in order to get commit privileges.

Checking Out the Repository

When getting started, you will need to checkout a copy of the latest codebase into a new directory. Create a directory called freewpc first, and switch into that: 

mkdir freewpc
cd freewpc

Then to checkout the code from the server, 

svn checkout http://www.oddchange.com/svn/public/freewpc/trunk

This will create a directory called trunk and populate it with the latest files in the repository.

Updating Your Working Copy

You can just do svn update, either at the root or in any subdirectory, to grab the latest versions of all the files. Most all Subversion commands work this way. You can also give a single filename, and it will only apply to that one file instead of the whole directory. 

svn update

Checking In Changes

Use the commit command to push your local changes back to the project server. You must have write privileges in order to perform this command.

svn commit

Checking the Status of All Files

Use the status command to get a summary of the status of each file and directory: 

svn status

The one-letter prefix in front of each file tells you the status: M=modified, ?=not in source control, C=in conflict!
